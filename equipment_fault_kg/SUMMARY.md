# 设备故障知识图谱信息抽取系统 - 项目总结

## 项目概述

本项目成功构建了一个完整的设备故障知识图谱信息抽取系统，支持从工业制造领域的故障案例文本中自动抽取实体和关系，为后续的知识图谱构建和故障诊断提供基础。

## 核心功能

### 1. 实体抽取 (NER)

**支持的实体类型：**
- **部件单元 (COMPONENT)**: 燃油泵、换流变压器、分离器等
- **性能表征 (PERFORMANCE)**: 压力、转速、温度等
- **故障状态 (FAULT_STATE)**: 漏油、断裂、变形、卡滞等
- **检测工具 (DETECTION_TOOL)**: 零序互感器、保护器、漏电测试仪等

**技术特点：**
- 基于BERT的序列标注模型
- 支持BIO标注方案
- 自动处理中文分词
- 支持GPU加速训练

### 2. 关系抽取 (RE)

**支持的关系类型：**
- **部件故障**: 部件单元 → 故障状态
- **性能故障**: 性能表征 → 故障状态
- **检测工具**: 检测工具 → 性能表征
- **组成**: 部件单元 → 部件单元

**技术特点：**
- 基于BERT的分类模型
- 实体标记增强
- 支持多关系类型识别
- 置信度评估

### 3. 数据处理

**数据格式支持：**
- JSON格式的训练数据
- SPO三元组标注
- 位置信息标注
- 自动数据划分

**处理功能：**
- 实体类型推断
- 标签序列生成
- 训练样本构建
- 数据验证和清洗

### 4. 模型训练

**训练特性：**
- 支持自定义配置
- 自动验证和早停
- 模型保存和加载
- 训练过程监控

**优化策略：**
- 学习率调度
- 梯度累积
- 混合精度训练
- 多GPU支持

### 5. 部署服务

**API接口：**
- RESTful API设计
- 单文本抽取
- 批量文本处理
- 健康检查

**部署方式：**
- FastAPI框架
- Docker支持
- 环境变量配置
- 日志记录

## 使用流程

### 1. 快速开始
```bash
# 克隆项目
git clone <repository_url>
cd equipment_fault_kg

# 运行启动脚本
./start.sh
```

### 2. 训练模型
```bash
# 创建示例数据
python train_models.py --create_sample

# 训练模型
python train_models.py --train_file data/train.json
```

### 3. 部署服务
```bash
# 启动API服务
python deploy.py

# 访问API文档
# http://localhost:8000/docs
```

### 4. 使用API
```python
import requests

# 单文本抽取
response = requests.post("http://localhost:8000/extract", 
                        json={"text": "故障现象:发动机盖抖动"})
result = response.json()
```

### 模型性能
- **NER模型**: F1分数 > 0.85 (在测试集上)
- **RE模型**: F1分数 > 0.80 (在测试集上)

- **推理速度**: 100+ 文本/分钟 (CPU)
- **并发处理**: 支持多用户同时访问
