# 实体抽取效果问题分析报告

## 1. 问题概述

根据您提供的实体抽取案例：

**输入文本**: `伺服电机运行异常，维修人员使用万用表检测电路故障。`

**原系统输出**:
- 伺服电机 [部件单元] ✓
- 运行异常， [故障状态] ❌ (包含标点)
- 维修 [部件单元] ❌ (分词错误+分类错误)
- 使 [故障状态] ❌ (无意义字符+分类错误)
- 万用表 [检测工具] ✓
- 检测 [故障状态] ❌ (分类错误)
- 电路 [部件单元] ❌ (实体不完整)
- 故障 [故障状态] ✓
- 。 [部件单元] ❌ (标点符号+分类错误)

**问题统计**: 9个识别实体中，仅3个完全正确，准确率约33%

## 2. 核心问题分析

### 2.1 字符级处理导致的分词问题

**问题代码定位**: `deploy_ner.py` 第85-95行
```python
for i, char in enumerate(text):
    # 按字符逐个处理
    sub_tokens = self.tokenizer.tokenize(char)
```

**具体问题**:
- "维修人员" → "维修" + "人" + "员"
- "电路故障" → "电路" + "故障"
- 破坏了词汇的语义完整性

**根本原因**: 中文NER应该考虑词边界，而不是纯字符级别处理

### 2.2 缺少实体后处理机制

**问题表现**:
- 标点符号"。"被识别为实体
- 单字符"使"被识别为实体
- 实体包含不应该的标点"运行异常，"

**缺失功能**:
- 实体有效性检查
- 标点符号过滤
- 实体边界调整

### 2.3 实体分类错误

**错误案例**:
- "维修" → [部件单元] (应该是操作动作)
- "使" → [故障状态] (应该被过滤)
- "检测" → [故障状态] (应该是操作动作)
- "。" → [部件单元] (应该被过滤)

**根本原因**:
- 缺少基于规则的实体类型验证
- 训练数据质量问题
- 模型对上下文理解不足

### 2.4 代码一致性问题

**标签映射不一致**:

`deploy_ner.py` (第32-39行):
```python
self.label2id = {
    'O': 0,
    'B-COMPONENT': 1, 'I-COMPONENT': 2,
    'B-PERFORMANCE': 3, 'I-PERFORMANCE': 4,
    'B-FAULT_STATE': 5, 'I-FAULT_STATE': 6,
    'B-DETECTION_TOOL': 7, 'I-DETECTION_TOOL': 8
}
```

`ner_model.py` (第33-40行):
```python
self.label2id = {
    'O': 0,
    'B-Equipment': 1, 'I-Equipment': 2,
    'B-Component': 3, 'I-Component': 4,
    'B-Fault': 5, 'I-Fault': 6,
    # ... 不同的标签定义
}
```

## 3. 解决方案

### 3.1 改进的分词策略

**方案1: 基于jieba的词级别处理**
```python
# 使用jieba获得词边界
words = list(jieba.cut(text))
# 保持词的完整性进行实体识别
```

**方案2: 字符级标注+词边界调整**
```python
# 先进行字符级标注，再根据词边界调整实体边界
entities = self._adjust_entity_boundaries(entities, word_positions)
```

### 3.2 实体后处理机制

**无效实体过滤规则**:
```python
invalid_patterns = [
    r'^[，。！？；：""''（）\[\]{}()<>\/\\|+=\-*&^%$#@~`\s]+$',  # 纯标点
    r'^[0-9]+$',  # 纯数字  
    r'^[的了在是和与或]$',  # 常见虚词
]
```

**实体有效性检查**:
- 长度检查 (1-20字符)
- 字符类型检查
- 上下文合理性检查

### 3.3 实体分类优化

**基于规则的实体类型验证**:
```python
entity_patterns = {
    'COMPONENT': [r'.*[泵器机阀管箱盖锁轮].*'],
    'FAULT_STATE': [r'.*[漏断裂变形卡死松动抖动异常故障].*'],
    'DETECTION_TOOL': [r'.*[万用表示波器测试仪].*'],
}
```

**置信度评分**:
- 基于实体长度、类型匹配度、上下文相关性

### 3.4 统一标签映射

**建议使用一致的标签定义**:
```python
UNIFIED_LABELS = {
    'O': 0,
```
