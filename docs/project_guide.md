# 装备制造故障知识图谱构建及应用案例剖析 - 项目指南

## 项目概述

本项目是一个完整的装备制造故障知识图谱构建及应用系统，旨在帮助学生学习和掌握：

1. **实体抽取技术**：基于深度学习的命名实体识别（NER）
2. **关系抽取技术**：实体间关系的自动识别和抽取
3. **知识图谱构建**：从文本数据构建结构化知识图谱
4. **本地大模型部署**：部署和使用开源大语言模型
5. **智能问答系统**：基于知识图谱的问答系统
6. **数据采集与处理**：多源数据采集、清洗和标注

## 项目架构

### 1. 技术栈

#### 后端技术
- **Python 3.8+**：主要开发语言
- **FastAPI**：现代、高性能的Web框架
- **Neo4j**：图数据库，存储知识图谱
- **MySQL**：关系型数据库，存储结构化数据
- **Redis**：缓存数据库，提高系统性能
- **PyTorch**：深度学习框架
- **transformers**：预训练模型库
- **spaCy**：自然语言处理库

#### 前端技术
- **Vue.js 3**：渐进式JavaScript框架
- **Element Plus**：Vue 3 UI组件库
- **ECharts**：数据可视化图表库
- **D3.js**：图谱可视化库

#### 数据库与存储
- **Neo4j**：图数据库，存储实体和关系
- **MySQL**：存储用户数据、诊断记录等
- **Redis**：缓存频繁访问的数据
- **Elasticsearch**：全文搜索引擎

### 3. 核心功能模块

#### 3.1 数据采集模块
- **PDF文档解析**：从技术手册中提取故障案例
- **网页爬虫**：爬取在线故障诊断资料
- **Excel数据导入**：处理结构化的故障数据
- **数据清洗**：去重、格式化、质量检查

#### 3.2 实体抽取模块
- **基于BERT的NER模型**：识别设备、故障现象、故障原因等实体
- **规则匹配**：使用正则表达式补充实体抽取
- **混合抽取**：结合深度学习和规则的混合方法

#### 3.3 关系抽取模块
- **基于预训练模型的关系分类**：识别实体间的关系类型
- **依存句法分析**：利用句法结构进行关系抽取
- **模式匹配**：基于预定义模式的关系抽取

#### 3.4 知识图谱构建模块
- **实体链接**：将提取的实体链接到知识库
- **关系建模**：构建实体间的关系网络
- **图谱优化**：去重、合并、质量评估

#### 3.5 故障诊断模块
- **基于规则的推理**：使用专家规则进行故障诊断
- **图谱查询**：在知识图谱中搜索相关信息
- **相似案例检索**：找到历史相似故障案例

#### 3.6 大模型服务模块
- **模型部署**：支持ChatGLM、Qwen等开源大模型
- **量化优化**：支持int4、int8量化以适应不同硬件
- **API服务**：提供标准化的文本生成API

## 学习路径

### 第一阶段：环境搭建与基础理解

#### 1. 环境准备
```bash
# 1. 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动数据库服务
docker-compose up -d neo4j mysql redis
```

#### 2. 理解项目结构
- 熟悉`backend/`目录：API服务、业务逻辑
- 了解`frontend/`目录：用户界面

## 常见问题与解决方案

### 1. 环境配置问题
**问题**：依赖安装失败
**解决**：
- 使用conda环境管理
- 检查Python版本兼容性
- 使用国内镜像源

### 2. 数据处理问题
**问题**：PDF解析失败
**解决**：
- 检查PDF文件格式
- 尝试不同的解析库
- 手动处理特殊格式

### 3. 模型训练问题
**问题**：显存不足
**解决**：
- 减少batch size
- 使用梯度累积
- 启用模型量化

### 4. 图数据库问题
**问题**：Neo4j连接失败
**解决**：
- 检查服务状态
- 验证连接参数
- 查看防火墙设置
