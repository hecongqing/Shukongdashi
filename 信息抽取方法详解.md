# 数控机床故障诊断知识图谱信息抽取方法详解

## 概述

这个项目是一个数控机床故障诊断知识图谱系统，其核心功能是从故障诊断文本中自动抽取**头实体（Subject）**、**尾实体（Object）** 和它们之间的**关系（Relation）**，构建结构化的知识三元组。

## 核心数据结构

### 关系数据类（Relation）
```python
@dataclass
class Relation:
    subject: str          # 头实体
    predicate: str        # 谓词/关系
    object: str          # 尾实体  
    relation_type: str   # 关系类型
    confidence: float    # 置信度
    source_text: str     # 源文本
    start_pos: int       # 开始位置
    end_pos: int         # 结束位置
```

## 三种信息抽取方法

### 1. 基于模式匹配的抽取（Pattern-based Extraction）

#### 原理
使用预定义的正则表达式模式来识别特定结构的关系表达。

#### 实现方式
- **关系模式定义**：针对不同关系类型定义具体的正则表达式
- **模式匹配**：在文本中搜索匹配的模式
- **实体提取**：从匹配的分组中提取头实体、谓词和尾实体

#### 具体示例

**故障-症状关系模式**：
```python
'fault_symptom': [
    r'([^，。；]*故障[^，。；]*)(导致|引起|造成|产生)([^，。；]*症状[^，。；]*)',
    r'([^，。；]*故障[^，。；]*)(出现|显示|表现)([^，。；]*症状[^，。；]*)'
]
```

**应用示例**：
- 输入文本：`"主轴故障导致加工精度下降"`
- 匹配模式：`([^，。；]*故障[^，。；]*)(导致)([^，。；]*)`
- 抽取结果：
  - **头实体**：主轴故障
  - **谓词**：导致
  - **尾实体**：加工精度下降
  - **关系类型**：fault_symptom

**故障-原因关系模式**：
```python
'fault_cause': [
    r'([^，。；]*故障[^，。；]*)(由|由于|因为|源于)([^，。；]*原因[^，。；]*)',
    r'([^，。；]*原因[^，。；]*)(导致|引起|造成)([^，。；]*故障[^，。；]*)'
]
```

### 2. 基于启发式规则的抽取（Heuristic-based Extraction）

#### 原理
通过分析句子结构和常见的关系谓词，使用启发式规则来识别实体间的关系。

#### 实现步骤
1. **句子分割**：将文本分割成独立的句子
2. **谓词识别**：在句子中寻找预定义的关系谓词
3. **实体定位**：根据谓词位置确定主语和宾语部分
4. **实体抽取**：从主语和宾语部分提取具体实体
5. **关系类型推断**：基于实体和谓词的语义推断关系类型

#### 常见关系谓词
```python
common_predicates = {
    '导致', '引起', '造成', '产生', '出现', '显示', '表现',
    '由', '由于', '因为', '源于', '解决方法', '解决方案', 
    '修复', '修理', '维修', '损坏', '故障', '问题', '影响',
    '使用', '需要', '采用', '伴随', '伴有', '排除', '解决'
}
```

#### 关系类型推断逻辑
```python
def _infer_relation_type(self, subject: str, predicate: str, object_entity: str) -> str:
    subject_lower = subject.lower()
    object_lower = object_entity.lower()
    predicate_lower = predicate.lower()
    
    # 故障相关关系
    if '故障' in subject_lower or '故障' in object_lower:
        if '症状' in subject_lower or '症状' in object_lower:
            return 'fault_symptom'    # 故障-症状关系
        elif '原因' in subject_lower or '原因' in object_lower:
            return 'fault_cause'      # 故障-原因关系
        elif '解决' in predicate_lower or '修复' in predicate_lower:
            return 'fault_solution'   # 故障-解决方法关系
    
    # 设备相关关系
    if '设备' in subject_lower and '故障' in object_lower:
        return 'equipment_fault'      # 设备-故障关系
```

#### 应用示例
- 输入文本：`"数控系统出现报警，原因是电源电压不稳定"`
- 处理流程：
  1. 发现谓词："出现"
  2. 分割句子：主语部分="数控系统"，宾语部分="报警"
  3. 抽取实体：头实体="数控系统"，尾实体="报警"
  4. 推断类型：equipment_fault

### 3. 基于已知实体的抽取（Entity-based Extraction）

#### 原理
利用预先识别的实体列表，在文本中寻找这些实体间的关系连接。

#### 实现流程
1. **实体定位**：在文本中找到已知实体的位置
2. **共现检查**：检查两个实体是否在同一句子中出现
3. **关系识别**：寻找连接两个实体的关系谓词
4. **方向确定**：根据实体和谓词的位置确定关系方向
5. **关系构建**：构建从头实体到尾实体的关系

#### 实现代码示例
```python
def _extract_by_entities(self, text: str, entities: List[str]) -> List[Relation]:
    relations = []
    
    for entity1 in entities:
        for entity2 in entities:
            if entity1 != entity2 and entity1 in text and entity2 in text:

                    if entity1 in sentence and entity2 in sentence:

                        for predicate in self.patterns.common_predicates:
                            if predicate in sentence:
                                # 确定实体顺序
                                entity1_pos = sentence.find(entity1)
                                entity2_pos = sentence.find(entity2)
                                predicate_pos = sentence.find(predicate)
                                
                                # 根据位置确定头尾实体
                                if entity1_pos < predicate_pos < entity2_pos:
                                    # entity1 -> predicate -> entity2
                                    relation = Relation(
                                        subject=entity1,      # 头实体
                                        predicate=predicate,  # 谓词
                                        object=entity2,       # 尾实体
                                        relation_type=self._infer_relation_type(entity1, predicate, entity2),
                                        confidence=0.7
                                    )
```

#### 应用示例
- 输入文本：`"主轴故障导致加工精度下降"`
- 已知实体：`["主轴", "加工精度", "轴承"]`
- 处理结果：
  - 发现"主轴"和"加工精度"共现
  - 找到连接谓词"导致"
  - 确定顺序：主轴 < 导致 < 加工精度
  - 构建关系：(主轴, 导致, 加工精度)

## 支持的关系类型

### 1. 故障相关关系
- **fault_symptom**：故障-症状关系
  - 示例：`主轴故障 → 导致 → 振动异常`
- **fault_cause**：故障-原因关系
  - 示例：`轴承磨损 → 导致 → 主轴故障`
- **fault_solution**：故障-解决方法关系
  - 示例：`主轴故障 → 解决方法 → 更换轴承`
- **fault_impact**：故障-影响关系
  - 示例：`主轴故障 → 影响 → 加工精度`

### 2. 设备相关关系
- **equipment_fault**：设备-故障关系
  - 示例：`数控机床 → 出现 → 主轴故障`
- **component_fault**：部件-故障关系
  - 示例：`轴承 → 故障 → 磨损严重`

### 3. 维修相关关系
- **maintenance_tool**：维修-工具关系
  - 示例：`维修 → 使用 → 万用表`
- **maintenance_personnel**：维修-人员关系
  - 示例：`维修 → 需要 → 专业技术人员`
- **maintenance_time**：维修-时间关系
  - 示例：`维修 → 耗时 → 2小时`

### 4. 检测相关关系
- **detection_method**：检测-方法关系
  - 示例：`故障检测 → 方法 → 振动分析`
- **detection_equipment**：检测-设备关系
  - 示例：`故障检测 → 使用 → 示波器`

## 关系验证机制

### 验证规则
1. **实体长度验证**：过滤过短（<2字符）或过长（>50字符）的实体
2. **实体质量验证**：过滤低质量实体（纯数字、纯英文等）
3. **关系语义验证**：验证关系的语义合理性
4. **重复检查**：检查和过滤重复关系
5. **置信度阈值**：过滤低置信度关系

### 置信度计算
- **模式匹配**：0.8（高置信度）
- **启发式方法**：0.6（中等置信度）
- **基于实体**：0.7（中高置信度）

## 系统特点

### 优势
1. **多方法融合**：结合模式匹配、启发式规则和实体约束
2. **领域特化**：专门针对数控机床故障诊断领域
3. **质量控制**：多重验证机制确保抽取质量
4. **可扩展性**：支持自定义关系模式和规则

### 应用场景
1. **知识图谱构建**：自动构建故障诊断知识图谱
2. **故障分析**：分析故障原因、症状和解决方案
3. **维修指导**：提供结构化的维修知识
4. **经验总结**：从大量文本中提取专家经验

这个信息抽取系统通过三种互补的方法，能够有效地从非结构化的故障诊断文本中抽取结构化的知识三元组，为数控机床故障诊断提供了强有力的知识支撑。
